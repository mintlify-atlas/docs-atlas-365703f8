Configuration properties are set via `rd_kafka_conf_set()` (global) or `rd_kafka_topic_conf_set()` (topic) **before** creating the client handle with `rd_kafka_new()`. Properties cannot be changed after the handle is created.

<Note>
  The **C/P** column indicates which client type the property applies to: `C` = Consumer, `P` = Producer, `*` = both.
</Note>

## Global configuration properties

The table below covers the most important global properties. For the full list, see [`CONFIGURATION.md`](https://github.com/confluentinc/librdkafka/blob/master/CONFIGURATION.md) in the librdkafka repository.

| Property | C/P | Default | Importance | Description |
|----------|-----|---------|------------|-------------|
| `bootstrap.servers` | `*` | _(none)_ | high | Initial list of brokers as a CSV of `host` or `host:port`. Alias for `metadata.broker.list`. |
| `client.id` | `*` | `rdkafka` | low | Client identifier string included in broker log messages. |
| `security.protocol` | `*` | `plaintext` | high | Protocol used to communicate with brokers. Options: `plaintext`, `ssl`, `sasl_plaintext`, `sasl_ssl`. |
| `group.id` | `C` | _(none)_ | high | Consumer group ID. All clients sharing the same `group.id` belong to the same group. |
| `enable.auto.commit` | `C` | `true` | high | Automatically and periodically commit offsets in the background. |
| `auto.offset.reset` | `C` | `largest` | high | Action when no initial offset exists: `earliest`, `latest`, or `error`. |
| `session.timeout.ms` | `C` | `45000` | high | Consumer group session and failure detection timeout (ms). |
| `heartbeat.interval.ms` | `C` | `3000` | low | Group session keepalive heartbeat interval (ms). |
| `max.poll.interval.ms` | `C` | `300000` | high | Maximum time between `rd_kafka_consumer_poll()` calls before the consumer is considered failed (ms). |
| `isolation.level` | `C` | `read_committed` | high | Transactional read mode: `read_committed` or `read_uncommitted`. |
| `statistics.interval.ms` | `*` | `0` | high | Stats emit interval (ms). `0` = disabled. Requires a `stats_cb` to be registered. |
| `debug` | `*` | _(none)_ | medium | Comma-separated list of debug contexts. E.g., `broker,topic,msg` for producer; `consumer,cgrp,topic,fetch` for consumer. |
| `socket.timeout.ms` | `*` | `60000` | low | Default timeout for network requests (ms). |
| `message.max.bytes` | `*` | `1000000` | medium | Maximum Kafka protocol request message size (bytes). |
| `max.in.flight` | `*` | `1000000` | low | Maximum number of in-flight requests per broker connection. Set to `5` when `enable.idempotence=true`. |
| `compression.codec` | `P` | `none` | medium | Compression codec for message sets. Options: `none`, `gzip`, `snappy`, `lz4`, `zstd`. |
| `linger.ms` | `P` | `5` | high | Delay (ms) to accumulate messages in the producer queue before sending. Alias for `queue.buffering.max.ms`. |
| `batch.num.messages` | `P` | `10000` | medium | Maximum number of messages batched in one MessageSet. |
| `batch.size` | `P` | `1000000` | medium | Maximum size (bytes) of all messages batched in one MessageSet, including protocol framing. |
| `enable.idempotence` | `P` | `false` | high | Enable idempotent producer. Automatically sets `acks=all`, `retries=INT32_MAX`, `max.in.flight=5`. |
| `transactional.id` | `P` | _(none)_ | high | Enables the transactional producer. Required for EOS. Requires broker >= 0.11.0. |
| `transaction.timeout.ms` | `P` | `60000` | medium | Maximum time the transaction coordinator waits for a status update before aborting the transaction (ms). |
| `message.timeout.ms` | `P` | `300000` | high | Local message delivery timeout (ms). `0` = infinite. Alias for `delivery.timeout.ms`. |
| `sasl.mechanism` | `*` | `GSSAPI` | high | SASL mechanism: `GSSAPI`, `PLAIN`, `SCRAM-SHA-256`, `SCRAM-SHA-512`, `OAUTHBEARER`. |
| `sasl.username` | `*` | _(none)_ | high | SASL username for `PLAIN` and `SCRAM` mechanisms. |
| `sasl.password` | `*` | _(none)_ | high | SASL password for `PLAIN` and `SCRAM` mechanisms. |
| `ssl.ca.location` | `*` | _(system)_ | low | Path to CA certificate(s) for verifying the broker's key. |
| `ssl.certificate.location` | `*` | _(none)_ | low | Path to client's public key (PEM) for mutual TLS authentication. |
| `ssl.key.location` | `*` | _(none)_ | low | Path to client's private key (PEM) for mutual TLS authentication. |

### Example: minimal producer

```c
rd_kafka_conf_t *conf = rd_kafka_conf_new();
char errstr[512];

rd_kafka_conf_set(conf, "bootstrap.servers", "broker1:9092,broker2:9092", errstr, sizeof(errstr));
rd_kafka_conf_set(conf, "security.protocol", "sasl_ssl", errstr, sizeof(errstr));
rd_kafka_conf_set(conf, "sasl.mechanism", "PLAIN", errstr, sizeof(errstr));
rd_kafka_conf_set(conf, "sasl.username", "myuser", errstr, sizeof(errstr));
rd_kafka_conf_set(conf, "sasl.password", "mypassword", errstr, sizeof(errstr));
rd_kafka_conf_set(conf, "enable.idempotence", "true", errstr, sizeof(errstr));

rd_kafka_t *rk = rd_kafka_new(RD_KAFKA_PRODUCER, conf, errstr, sizeof(errstr));
```

<Tip>
  After calling `rd_kafka_new()`, the `conf` object is owned by the handle and must not be freed by the application.
</Tip>

## Topic configuration properties

Topic properties are set via `rd_kafka_topic_conf_set()` before creating a topic handle, or via the global `default_topic_conf` property.

| Property | C/P | Default | Importance | Description |
|----------|-----|---------|------------|-------------|
| `acks` | `P` | `-1` | high | Number of broker acknowledgements required. `0` = none, `1` = leader only, `-1`/`all` = all ISRs. Alias for `request.required.acks`. |
| `request.required.acks` | `P` | `-1` | high | Same as `acks`. |
| `message.timeout.ms` | `P` | `300000` | high | Local message delivery timeout (ms). Delivery fails if not acknowledged within this window. |
| `compression.codec` | `P` | `inherit` | high | Per-topic compression codec. `inherit` uses the global `compression.codec` setting. |
| `auto.offset.reset` | `C` | `largest` | high | Offset reset policy: `earliest`, `latest`, or `error`. |

<Warning>
  Setting `acks=0` disables delivery guarantees entirely — the broker will not confirm receipt. Use `-1` (all) for maximum durability, especially with `enable.idempotence=true`.
</Warning>
